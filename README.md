# Hermes

**Hermes** - тестирующая система, предназначенная для проверки решений пользователей по задачам.
Система позволяет создавать задачи с тестами, принимать решения, выполнять код в изолированной среде и выставлять вердикт.

**Текущий статус** Реализована базовая модель задач и тестов(позже будет миграция на более совершенные модели хранения задач), модель решения по задаче, REST API для управления ими.

## API (текущая версия)

Все эндпоинты имеют префикс `/api/v1`

### Задачи
| Метод | Путь | Описание |
|-------|------|----------|
| `POST`   | `/tasks`           | Создать новую задачу        |
| `GET`    | `/tasks`           | Получить все задачи         |
| `GET`    | `/tasks/:id`       | Получить задачу по ID       |
| `DELETE` | `/tasks/:id`       | Удалить задачу по ID        |
| `PUT`    | `/tasks/:id`       | Обновить задачу[^1]         |
| `POST`   | `/submissions`     | Отправить решение по задаче |
| `GET`    | `/submissions/:id` | Получить решение по задаче  |


[^1]: Если в переданной(новой) задаче не будет тестов, они останутся прежними.

## Архитектура
1. HTTP API(Gin) - прием и обработка запросов от пользователей.
2. Хендлеры - валидация и маршрутизация к сервисам.
3. Сервисный слой - управление задачами, решениями, проверкой.
4. Репозиторий(GORM) - работа с PostgreSQL(позже возможна миграция на другую БД).
5. Очередь(планируется) - асинхронная проверка решений.
6. Исполнитель/runner(планируется) - изолированный запуск пользовательского кода(Docker или иные контейнеры).
